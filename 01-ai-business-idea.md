# PEHLA DELIVERABLE (Kal nahi, abhi)

Main chahta hoon ke hum coding se pehle **Product Requirement Document (PRD)** finalize karein.

Ye 30–40 pages ka document hoga, jisme:

* Problem statement
* Target customer
* User personas
* User journeys
* Features (V1, V2, V3)
* Database entities
* API boundaries
* AI agent responsibilities
* Success metrics
* Pricing
* Competitive analysis
* Launch plan

**Ye document hamara "constitution" hoga.**

Uske baad Codex ko kabhi confuse nahi hona padega.

## Aur ek commitment meri taraf se

Main aapko unnecessarily code generate karke nahi dunga.

Mera pehla instinct hamesha ye hoga:

1. Kya ye feature V1 ke liye zaroori hai?
2. Iska simplest implementation kya hai?
3. Isse customer ko measurable value kitni milegi?
4. Kya AI is process ko reliably automate kar sakti hai?

Agar in 4 sawalon ke jawab strong honge, tabhi implementation phase mein jayenge.

**Ab se hamara objective sirf software banana nahi hai. Hamara objective hai pehla paying customer hasil karna.** Har decision isi goal ke against measure hoga.
